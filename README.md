## 📖 About the Project

**SoundScan** is an AI-powered music authenticity detection system built as part of our undergraduate thesis on **Hybrid CNN-based audio analysis**. The system determines whether a given piece of music was **generated by AI** or **performed by a human**, with explainable results backed by acoustic and neural analysis.

Our model — the **HybridASTDetector** — combines a pretrained **Audio Spectrogram Transformer (AST)** backbone with custom **CNN blocks**, fusing deep neural representations with handcrafted acoustic features (beat regularity, harmonic ratio, spectral flatness, pitch stability, dynamic range, and more) to produce a final weighted verdict.

The web interface (**SoundScan**) lets users upload audio files and receive a real-time breakdown including:

- ✅ AI vs. Human probability score
- 🎸 Instrument detection
- 🎼 Genre classification (GTZAN + Discogs)
- 🧠 Explainable AI (XAI) via Grad-CAM
- 📊 Acoustic deep analysis per segment

---

## 🧠 How It Works

The detection pipeline works in four stages:

1. **Audio Ingestion** — Uploaded audio (MP3, WAV, FLAC, OGG, M4A, AAC) is converted to a standardized 16 kHz mono WAV.
2. **Feature Extraction** — Mel spectrograms are computed alongside acoustic features such as beat regularity, zero-crossing rate, noise floor, and codec compression markers.
3. **Hybrid Inference** — The HybridASTDetector runs inference on weighted audio segments (intro, verse, chorus, bridge, outro). Neural output (85%) is blended with acoustic scoring (15%) for a final AI probability.
4. **Explainability** — Grad-CAM highlights which spectrogram regions triggered the verdict, and LIME generates interpretable segment-level explanations.

## 🚀 How to Run

### 1. Clone the Repository

```bash
git clone https://github.com/your-username/hybrid-cnn-soundscan.git
cd hybrid-cnn-soundscan
```

### 2. Create and Activate a Virtual Environment

```bash
# Create environment
python -m venv venv

# Activate on Windows
venv\Scripts\activate

# Activate on macOS/Linux
source venv/bin/activate
```

### 3. Install Dependencies

```bash
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu118
pip install flask flask-cors transformers soundfile numpy
```

> 💡 If you don't have a CUDA GPU, replace `cu118` with `cpu` in the PyTorch install URL.

### 4. Add the Trained Model Checkpoint

Place your trained model file in the root directory:

```
hybrid-cnn-soundscan/
├── app.py
├── index.html
├── best_model.pth        ← required
└── ...
```

> The model checkpoint (`best_model.pth`) must be present for the server to start. Contact the authors if you need access to the pretrained weights.

### 5. Install FFmpeg

FFmpeg is required to convert non-WAV audio formats.

- **Windows**: Download from [ffmpeg.org](https://ffmpeg.org/download.html) and add to PATH
- **macOS**: `brew install ffmpeg`
- **Linux**: `sudo apt install ffmpeg`

### 6. Run the Backend Server

```bash
python app.py
```

You should see:

```
* Running on http://0.0.0.0:5000
```

### 7. Open the Frontend

Open `index.html` directly in your browser, **or** navigate to:

```
http://localhost:5000
```

Upload any audio file (MP3, WAV, FLAC, OGG, M4A, or AAC) and click **Analyze** to get your result.
